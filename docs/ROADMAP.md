# Roadmap

## Phase 1 — Stabilize the scaffold
- restore missing source files and routes
- make package, tsconfig, env, and verification scripts real
- ensure the project builds and verifies locally

## Phase 2 — Harden the parent control surface
- realtime dashboard updates for child profiles, approvals, welfare logs, and incidents
- approval resolution UX in dashboard and child detail screens
- better child detail visibility into welfare and gating state

## Phase 3 — Operationalize safety
- welfare-first learning filters
- green-only embodiment dispatch
- milestone review blocks when welfare is red
- module guards that require approval and, when needed, green welfare

## Phase 4 — Expand perception
- Dot Matrix depth-aware fusion
- candidate extraction from activation maps
- perception pipeline abstraction
- simulation harness for future camera and LiDAR work
